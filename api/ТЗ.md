## 1. Тестовое задание для backend (Go)

### Задача

Сделать сервис «Question Bank» со следующими возможностями:

1. **Сущности**

    * `Question`
        * id
        * role (строка: например, `backend_junior`, `frontend_junior`)
        * topic (строка)
        * type (`single_choice`, `multi_choice`, `text`)
        * difficulty (целое 1–5)
        * text (формулировка вопроса)
    * `Option` (для choice-вопросов)

        * id
        * questionId
        * text
        * isCorrect (bool)
    * `TestTemplate`

        * id
        * name
        * role
        * purpose (`skills_assessment`, `mock_interview`)
    * `TestTemplateQuestion`

        * templateId
        * questionId
        * order

2. **API (всё JSON, REST, без авторизации — можно добавить, но не обязательно)**

   CRUD для вопросов:

    * `POST /questions` — создать вопрос c опциями (если type = choice).
    * `GET /questions/{id}` — получить вопрос с опциями.
    * `GET /questions` — список с фильтрами: `role`, `topic`, `difficulty`.
    * `PUT /questions/{id}` — обновить вопрос и опции.
    * `DELETE /questions/{id}` — удалить вопрос.

   CRUD для шаблонов:

    * `POST /templates` — создать шаблон: роль, purpose, набор questionId с order.
    * `GET /templates/{id}` — получить шаблон с вопросами (минимум id + text).
    * `GET /templates` — список шаблонов (фильтры по роли и purpose).
    * `PUT /templates/{id}` — обновить.
    * `DELETE /templates/{id}` — удалить.

   Выдача теста:

    * `POST /tests/instantiate`
      Вход:

      ```json
      {
        "templateId": "…"
      }
      
      Выход:
 
      
      {
        "sessionId": "…",
        "questions": [
          {
            "id": "…",
            "text": "…",
            "type": "single_choice",
            "options": [ ... ]
          }
        ]
      }

3. Хранение

    * Любой подходящий ORM/driver, БД — PostgreSQL.
    * Миграции обязательно.

4. Качество/обязательные вещи

    * Структура проекта: разнести хендлеры, доменную логику, доступ к БД.
    * Валидация входных данных (обязательные поля, ограничения).
    * Нормальная обработка ошибок (коды HTTP, сообщение).
    * Несколько unit-тестов/интеграционных тестов:

        * хотя бы на логику создания/получения вопросов и шаблонов.

5. Инфраструктура

    * Dockerfile для сервиса.
    * docker-compose для сервиса + PostgreSQL.
    * README:

        * как запустить локально;
        * примеры запросов (curl или httpie).
```
